## 🚀 사용 방법

### 고객용 견적 요청 페이지

1. **프린터 기종 입력**
   - 예: 엡손1390, HP DesignJet 등
   - 모르면 '모름' 입력

2. **고장 증상 선택**
   - 잉크 막힘, 작동 불량, 전원 불량, 기타

3. **사진/영상 업로드**
   - 사진: 최대 10장
   - 영상: 10초 미만 (20MB 이하)

4. **방문 주소 입력**
   - 🔍 주소 검색 버튼 클릭
   - 다음 주소 검색 팝업에서 주소 선택
   - **자동으로 거리 계산 및 출장비 합산**

5. **방문 희망 날짜/시간 선택**
   - 달력에서 날짜 클릭
   - 시간대 선택 (오전/오후/협의)

6. **고객 정보 입력**
   - 이름, 전화번호

7. **견적 요청 제출**
   - 개인정보 동의 체크
   - 제출 버튼 클릭
   - **이메일 전송 + DB 저장 완료!**

5. **출장비 설정**
   - 우측 패널에서 기본 출장비 설정
   - 저장 버튼 클릭

6. **예약 불가 날짜 관리**
   - 달력에서 날짜 클릭 → 예약 불가로 설정
   - 저장 버튼 클릭

## 🗄️ 데이터베이스 구조

### `printer_repair_requests` 테이블

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | BIGINT | 자동 증가 ID |
| `created_at` | TIMESTAMPTZ | 생성 시각 (자동) |
| `printer_model` | TEXT | 프린터 기종 |
| `symptoms` | TEXT[] | 고장 증상 배열 |
| `description` | TEXT | 상세 설명 |
| `customer_name` | TEXT | 고객명 |
| `customer_phone` | TEXT | 전화번호 |
| `customer_address` | TEXT | 방문 주소 |
| `visit_date` | TEXT | 방문 날짜 |
| `visit_time` | TEXT | 방문 시간 |
| `total_price` | NUMERIC | 총 출장비 |
| `image_urls` | TEXT[] | 이미지 URL 배열 |

### `blocked_dates` 테이블

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | UUID | 자동 생성 ID |
| `date` | DATE | 예약 불가 날짜 |
| `created_at` | TIMESTAMPTZ | 생성 시각 |

## 🔧 주요 기능 흐름

### 1. 고객이 견적 요청 제출 시

```
고객 입력 → 유효성 검증 → 이미지 업로드 (Supabase Storage) 
    ↓
Public URL 생성 → DB 저장 (printer_repair_requests) 
    ↓
EmailJS 이메일 전송 → 성공 메시지 표시 → 메인 페이지 이동
```

### 2. 관리자가 대시보드 확인 시

```
페이지 로드 → DB 조회 (printer_repair_requests) 
    ↓
데이터 매핑 → 화면 표시 
    ↓
Realtime 구독 → 새 데이터 자동 업데이트
```

### 3. 주소 검색 및 거리 계산

```
주소 검색 버튼 클릭 → 다음 주소 검색 팝업 
    ↓
주소 선택 → 카카오 Geocoder로 좌표 변환 
    ↓
본사 좌표와 거리 계산 (하버사인 공식) 
    ↓
출장비 자동 계산 → 폼 데이터 업데이트 → 사용자에게 알림
```
